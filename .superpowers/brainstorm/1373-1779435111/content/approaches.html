Craft Batch Planner — Approaches

How should the planner help you pick a diversified, profitable crafting batch?

A

Smart Batch Builder

You set a budget and batch size (e.g. "5M gil, 8 items"). The planner runs your existing craft flip engine, then applies a diversification algorithm: it picks the top item, then penalizes same-category items before picking the next, repeating until the batch is full or budget is spent.

Output: a ready-to-go batch with total cost, expected revenue, and a one-click "Send all to Shopping List" button.

Pros

  • One click → optimized batch
  • Diversification is automatic
  • Budget-aware from the start
  • Builds on existing craft flip logic

Cons

  • Less manual control over what's in the batch
  • Algorithm picks may not match your intuition
B

Interactive Batch Cart

The craft flip results table gets a "+ Add to Batch" button per row. A sticky bottom panel shows your running batch: total cost, expected profit, category breakdown, and budget remaining. Category badges go red when you're over-concentrating.

You browse and pick manually, but the UI gives you live feedback on diversification and budget.

Pros

  • Full control — you pick every item
  • Live budget + diversity feedback
  • Familiar craft flip table UX

Cons

  • More manual work each session
  • Doesn't suggest — only warns
C

Hybrid: Auto-suggest + Edit

Combines A and B. The planner auto-generates a recommended batch (like A), then shows it in an editable table where you can swap items in/out (like B). Removing an item re-runs the algorithm to suggest a replacement. Budget slider + category balance bar update live.

Best of both: fast start with full override control.

Pros

  • Fast start — useful batch in seconds
  • Full override when your gut disagrees
  • Budget + diversity always visible
  • Swap suggestions keep the batch optimized

Cons

  • Most complex to build
  • Algorithm needs to be good enough that suggestions feel helpful, not random

My recommendation: C (Hybrid)

You said you want help deciding and you like to diversify manually. The hybrid gives you a smart starting point without taking away control. We can build A first as the core engine, then layer on the edit UI — so it's incremental, not all-or-nothing.
